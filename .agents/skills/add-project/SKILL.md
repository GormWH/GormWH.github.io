# Add a New Project Entry

Use this when adding a portfolio project under `projects/`.

## Steps

### 1. Create a markdown file

Create `projects/{slug}.md` with frontmatter matching `src/content.config.ts`:

```markdown
---
title: "Project Title"
pubDate: 2026-03-26
description: "One-sentence summary of what was built and why it matters."
author: "SuHong Park"
image:
  url: "https://example.com/image.webp"
  alt: "Project screenshot description"
tags: ["backend", "system-design", "realtime"]
---

## Context
What problem did this solve?

## Approach
How did you design the system and why?

## Outcome
What changed, what was learned, and what is next?
```

### 2. Ensure required schema fields exist

Required fields:
- `title`
- `pubDate` (date)
- `description`
- `author`
- `image.url`
- `image.alt`
- `tags` (array of strings)

### 3. Write outcome-focused content

Prioritize:
- constraints and trade-offs
- architecture choices
- measurable outcomes or clear lessons

### 4. Verify listing and detail routes

The project should appear automatically in:
- `/projects`
- `/projects/{slug}` (generated by `src/pages/projects/[...slug].astro`)

### 5. Update docs

- Add route example to `docs/generated/route-map.md` if needed
- If content conventions changed, update `docs/guides/content-authoring.md`

## Verification

Run:

```sh
npm run build
```

Fix any schema or content rendering errors before finishing.
